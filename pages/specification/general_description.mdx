
# 2. General Description

## 2.1 Interaction of the Product
The application will interact with clients (users), couriers, and administrators through various interfaces:
- **Clients** will place orders and track them.
- **Couriers** will accept orders and update statuses.
- **Administrators** will manage users, couriers, and orders.

## 2.2 Product Functions
1. User registration and authentication.
2. Managing delivery orders (create, view, update, delete).
3. Managing couriers (assign couriers to orders).
4. Displaying order and courier statistics.

## 2.3 User Characteristics
Users will be of the following types:
- **Clients** – can create orders and track them.
- **Couriers** – can accept orders and update their statuses.
- **Administrators** – can manage all aspects of the system, including users and orders.

## 2.4 Constraints
- The system should support both web and mobile versions of the application.
- The implementation should use **Nuxt.js** for the frontend and **Node.js** for the backend.

## 2.5 Assumptions and Dependencies
- The system will be deployed on a **Node.js**-based server.
- **JWT** will be used for authentication.
