
# 3. Detailed Requirements

## 3.1 External Interface Requirements

### 3.1.1 User Interfaces
1. **Web interface** using **Nuxt.js**.
2. **REST API** for interaction with the frontend.
3. **JWT** for authentication and authorization.

### 3.1.2 Hardware Interfaces
The application will run on a server with internet access.

### 3.1.3 Software Interfaces
1. **OpenAPI** for API documentation.
2. **Zudoku** for viewing the API documentation.
3. **Nuxt.js** for creating the client-side web application.

### 3.1.4 Interaction Interfaces
- Interaction between the client and server via **REST API**.
- **JSON** format for data transmission.

## 3.2 Functional Requirements

1. **Registration and Authentication**:
   - Users should be able to register and log in using **JWT**.

2. **Order Management**:
   - Admins can create, update, view, and delete orders.
   - Couriers can accept orders and update their statuses.

3. **Courier Management**:
   - Admins can assign couriers to orders.

## 3.3 Performance Requirements
- API response time should be less than 1 second.
- The system should support up to 1000 concurrent users.

## 3.4 Design Constraints
- Use of **Nuxt.js** for the frontend.
- The server should be based on **Node.js**.
- **OpenAPI** and **Zudoku** for API documentation.

## 3.5 Non-functional Requirements
- **Reliability**: The system should ensure 99.9% uptime.
- **Security**: Authentication via **JWT**, protection against CSRF.
- **Scalability**: The system should be scalable to accommodate growing user numbers.

## 3.6 Other Requirements
- Support for multiple languages.
- Logging of user actions and errors.
