
# 1. Introduction

## 1.1 Purpose
The purpose of this technical specification (TS) is to develop a web application for delivery management using **Nuxt.js**, which will work with the backend API defined in the **OpenAPI** specification.

## 1.2 Scope
The web application will be used for managing delivery orders, interacting with couriers, and tracking order statuses. The system should support user registration, authentication, and session management via JWT tokens.

## 1.3 Definitions, Acronyms, and Abbreviations
- **API** – Application Programming Interface.
- **JWT** – JSON Web Token.
- **Nuxt.js** – Framework for developing server-side and client-side applications using Vue.js.
- **Zudoku** – Tool for documenting OpenAPI specifications.
- **OpenAPI** – Specification for building RESTful APIs.

## 1.4 References
- OpenAPI Specification: [https://swagger.io/specification/](https://swagger.io/specification/)
- Nuxt.js Documentation: [https://nuxtjs.org/docs/](https://nuxtjs.org/docs/)
- Zudoku: [https://zudoku.dev/](https://zudoku.dev/)

## 1.5 Overview
This project involves creating the backend API, which will serve the **Nuxt.js** frontend. An API documentation will also be created using **OpenAPI**, which will be displayed via **Zudoku**.
